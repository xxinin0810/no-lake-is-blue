# 项目建档总结：《没有湖泊是蓝色的》WebVR 化探索

> 建档时间：2026-04-16
> 状态：规划阶段

---

## 1. 项目核心定位

### 内容属性
- 改编自《庄周梦蝶》，具有极强的情绪张力与诗意表达

### 视觉风格
- 追求极简、梦幻、风格化（Stylized）的 Low-Poly 艺术表现，而非极致写实
- 强调从"黑白"到"色彩蔓延"的视觉反差

### 商业定位
- 作为彰显个人艺术审美与技术整合能力的独立数字作品/数字名片
- 暂不作为 B2B 商业变现项目

---

## 2. 核心技术栈选择（AI Vibe-coding 驱动）

### 实时渲染路线
- **A-Frame / Three.js**
- 底层为纯文本/声明式代码，极其适合通过 AI（大语言模型）辅助编写核心交互逻辑
- 支持定制化的 GLSL Shader（如色彩侵染特效）

### 全景视频路线
- 若追求影视级画质，可利用 3D 软件预渲染高质量的立体并排（SBS）视频格式
- 在注入全景空间元数据后，通过 Web 端轻量化播放器实现沉浸式体验

### 引擎对比
- 放弃重度的 Unity 打包工作流
- 以换取"点开链接即看"的零门槛传播优势
- 参考 Wonderland Engine 等前沿框架的性能天花板

---

## 3. 架构设计与性能优化

### 多页面应用 (MPA) 架构
将不同的梦境场景分散挂载在独立的 HTML 页面上：
- 黑白房间
- 湖泊孤舟
- 其他梦境场景...

### 优势
1. **内存管理**：利用浏览器的机制在页面跳转时强制销毁内存，彻底解决移动端 VR（如 PICO 4）的显存泄漏与掉帧问题

2. **叙事融合**：将页面跳转时的"黑屏加载"巧妙转化为主角梦境切换时的"惊醒"与"失重"转场

3. **服务器部署**：采用极低成本方案
   - 轻量级服务器仅用于托管 HTML/JS 代码
   - 所有高内存的 3D 模型与贴图资源全部交由 CDN（内容分发网络）进行高速下发
   - 最终渲染算力完全依赖体验者头显（骁龙 XR2芯片）的本地性能

## 4. 技术要点备忘

### React Three.js 在 Strict Mode 下的常见问题
- `useRef` 的值在组件卸载后不会重置
- `isInitializedRef.current = true` 会阻止第二次初始化
- **解决方案**：
  1. 在 cleanup 中重置标志位
  2. 确保所有资源完全清理
  3. 使用 try-catch 捕获初始化错误
  4. 使用 setTimeout 延迟初始化避免竞态
